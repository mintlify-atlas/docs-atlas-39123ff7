# Quick Start

This guide will walk you through creating a simple particle system with Phenomenon. You'll have thousands of animated particles rendering at 60fps in just a few minutes.

## Prerequisites

Make sure you have:

- Phenomenon installed (`npm install phenomenon`)
- A `<canvas>` element in your HTML
- Basic understanding of JavaScript and WebGL concepts

## Basic Setup

<Steps>
  <Step title="Create the renderer">
    First, import Phenomenon and create a renderer instance:

    ```javascript
    import Phenomenon from 'phenomenon';

    const phenomenon = new Phenomenon({
      settings: {
        devicePixelRatio: 1,
        position: { x: 0, y: 0, z: 2 }
      }
    });
    ```

    <Note>
      The renderer automatically finds the first `<canvas>` element. You can specify a custom canvas with the `canvas` option.
    </Note>
  </Step>

  <Step title="Define particle attributes">
    Attributes are values stored on the GPU for each particle. Let's create position and color attributes:

    ```javascript
    const attributes = [
      {
        name: 'aPosition',
        data: (i) => [
          Math.random() * 2 - 1,  // x: -1 to 1
          Math.random() * 2 - 1,  // y: -1 to 1
          Math.random() * 2 - 1   // z: -1 to 1
        ],
        size: 3
      },
      {
        name: 'aColor',
        data: () => [
          Math.random(),  // r: 0 to 1
          Math.random(),  // g: 0 to 1
          Math.random()   // b: 0 to 1
        ],
        size: 3
      }
    ];
    ```

    <Tip>
      The `data` function is called once for each particle. Use the index `i` to create unique values.
    </Tip>
  </Step>

  <Step title="Create shaders">
    Write vertex and fragment shaders to position and color your particles:

    <CodeGroup>
      ```glsl Vertex Shader
      attribute vec3 aPosition;
      attribute vec3 aColor;

      uniform mat4 uProjectionMatrix;
      uniform mat4 uModelMatrix;
      uniform mat4 uViewMatrix;

      varying vec3 vColor;

      void main(){
        gl_Position = uProjectionMatrix * uModelMatrix * uViewMatrix * vec4(aPosition, 1.0);
        gl_PointSize = 2.0;
        vColor = aColor;
      }
      ```

      ```glsl Fragment Shader
      precision mediump float;

      varying vec3 vColor;

      void main(){
        gl_FragColor = vec4(vColor, 1.0);
      }
      ```
    </CodeGroup>
  </Step>

  <Step title="Add the instance">
    Combine everything and add the particle system to the renderer:

    ```javascript
    phenomenon.add('particles', {
      attributes,
      multiplier: 10000,  // Create 10,000 particles
      vertex: `
        attribute vec3 aPosition;
        attribute vec3 aColor;
        uniform mat4 uProjectionMatrix;
        uniform mat4 uModelMatrix;
        uniform mat4 uViewMatrix;
        varying vec3 vColor;

        void main(){
          gl_Position = uProjectionMatrix * uModelMatrix * uViewMatrix * vec4(aPosition, 1.0);
          gl_PointSize = 2.0;
          vColor = aColor;
        }
      `,
      fragment: `
        precision mediump float;
        varying vec3 vColor;

        void main(){
          gl_FragColor = vec4(vColor, 1.0);
        }
      `
    });
    ```
  </Step>
</Steps>

## Complete Example

Here's the complete code for a basic particle system:

```javascript
import Phenomenon from 'phenomenon';

// Create renderer
const phenomenon = new Phenomenon({
  settings: {
    devicePixelRatio: 1,
    position: { x: 0, y: 0, z: 2 }
  }
});

// Define attributes
const attributes = [
  {
    name: 'aPosition',
    data: (i) => [
      Math.random() * 2 - 1,
      Math.random() * 2 - 1,
      Math.random() * 2 - 1
    ],
    size: 3
  },
  {
    name: 'aColor',
    data: () => [
      Math.random(),
      Math.random(),
      Math.random()
    ],
    size: 3
  }
];

// Add particle system
phenomenon.add('particles', {
  attributes,
  multiplier: 10000,
  vertex: `
    attribute vec3 aPosition;
    attribute vec3 aColor;
    uniform mat4 uProjectionMatrix;
    uniform mat4 uModelMatrix;
    uniform mat4 uViewMatrix;
    varying vec3 vColor;

    void main(){
      gl_Position = uProjectionMatrix * uModelMatrix * uViewMatrix * vec4(aPosition, 1.0);
      gl_PointSize = 2.0;
      vColor = aColor;
    }
  `,
  fragment: `
    precision mediump float;
    varying vec3 vColor;

    void main(){
      gl_FragColor = vec4(vColor, 1.0);
    }
  `
});
```

## Adding Animation

Let's make the particles rotate by adding an `onRender` hook:

```javascript
phenomenon.add('particles', {
  attributes,
  multiplier: 10000,
  vertex: vertexShader,
  fragment: fragmentShader,
  uniforms: {
    uTime: {
      type: 'float',
      value: 0.0
    }
  },
  onRender: (instance) => {
    // Increment time uniform each frame
    instance.uniforms.uTime.value += 0.01;
  }
});
```

Then update your vertex shader to use the time uniform:

```glsl
attribute vec3 aPosition;
uniform float uTime;
uniform mat4 uProjectionMatrix;
uniform mat4 uModelMatrix;
uniform mat4 uViewMatrix;

void main(){
  // Rotate around Y axis
  float angle = uTime;
  vec3 rotated = vec3(
    aPosition.x * cos(angle) - aPosition.z * sin(angle),
    aPosition.y,
    aPosition.x * sin(angle) + aPosition.z * cos(angle)
  );
  gl_Position = uProjectionMatrix * uModelMatrix * uViewMatrix * vec4(rotated, 1.0);
  gl_PointSize = 2.0;
}
```

<Warning>
  The `onRender` hook is called every frame. Keep the logic lightweight to maintain 60fps.
</Warning>

## Advanced Example: Particle Transition

Here's a more advanced example showing particles transitioning between positions:

```javascript
const multiplier = 5000;
const duration = 0.6;

const attributes = [
  {
    name: 'aPositionStart',
    data: () => [
      Math.random() - 0.5,
      Math.random() - 0.5,
      Math.random() - 0.5
    ],
    size: 3
  },
  {
    name: 'aPositionEnd',
    data: () => [
      Math.random() - 0.5,
      Math.random() - 0.5,
      Math.random() - 0.5
    ],
    size: 3
  },
  {
    name: 'aOffset',
    data: (i) => [i * ((1 - duration) / (multiplier - 1))],
    size: 1
  }
];

const uniforms = {
  uProgress: {
    type: 'float',
    value: 0.0
  }
};

const vertex = `
  attribute vec3 aPositionStart;
  attribute vec3 aPositionEnd;
  attribute float aOffset;
  uniform float uProgress;
  uniform mat4 uProjectionMatrix;
  uniform mat4 uModelMatrix;
  uniform mat4 uViewMatrix;

  float easeInOutQuint(float t){
    return t < 0.5 ? 16.0 * t * t * t * t * t : 1.0 + 16.0 * (--t) * t * t * t * t;
  }

  void main(){
    float tProgress = easeInOutQuint(min(1.0, max(0.0, (uProgress - aOffset)) / ${duration}));
    vec3 newPosition = mix(aPositionStart, aPositionEnd, tProgress);
    gl_Position = uProjectionMatrix * uModelMatrix * uViewMatrix * vec4(newPosition, 1.0);
    gl_PointSize = 2.0;
  }
`;

phenomenon.add('transition', {
  attributes,
  multiplier,
  vertex,
  fragment: `
    precision mediump float;
    void main(){
      gl_FragColor = vec4(1.0, 1.0, 1.0, 1.0);
    }
  `,
  uniforms,
  onRender: (instance) => {
    instance.uniforms.uProgress.value += 0.01;
    if (instance.uniforms.uProgress.value > 1.0) {
      instance.uniforms.uProgress.value = 0.0;
    }
  }
});
```

<Tip>
  This example uses the `mix()` GLSL function to interpolate between start and end positions, creating smooth transitions.
</Tip>
