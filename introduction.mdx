# Welcome to Phenomenon

Phenomenon is a very small, low-level WebGL library that provides the essentials to deliver a high performance experience. Its core functionality is built around the idea of moving **millions of particles** around using the power of the GPU.

<Note>
  At just 2kB gzipped with zero dependencies, Phenomenon delivers enterprise-grade WebGL particle rendering without the bloat.
</Note>

## Key Features

- **Small in size** - No dependencies, minimal footprint
- **GPU based** - Hardware acceleration for maximum performance
- **Low-level & configurable** - Direct access to WebGL primitives
- **Dynamic instances** - Add and destroy particle systems on the fly
- **Dynamic attributes** - Update particle properties in real-time
- **TypeScript support** - Full type definitions included

## Performance

Phenomenon leverages the GPU to move millions of particles efficiently. By using WebGL buffers and custom shaders, you can create complex particle systems that run at a smooth 60fps even on modest hardware.

<Tip>
  For optimal performance, use a higher `multiplier` value with fewer instances rather than many instances with low multipliers.
</Tip>

## Architecture

Phenomenon consists of two main classes:

### Renderer

The main class that manages the WebGL context, canvas, and all particle instances. It handles:

- Setting up the WebGL context
- Managing projection, view, and model matrices
- Orchestrating render loops
- Sharing uniforms across instances

### Instance

Represents a single particle system or drawable object. Each instance has:

- Custom vertex and fragment shaders
- Attributes (data stored on the GPU)
- Uniforms (values that can be updated each frame)
- Geometry data (vertices and normals)
- A multiplier for creating many particles from one definition
